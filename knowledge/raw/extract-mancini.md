## EXTRACTION RULES (Quick Ref Version)

### 1. Setups - Ultra-Compact
```
Failed Breakdowns:
* Code block format only (no paragraphs)
* 8 lines per setup: Low/Flush/Accept/Entry/Stop/T1/T2/Run
* One warning line max
* One context sentence

Level Reclaims:
* Same code block format
* Fewer details than FB (secondary priority)

Max setups: 3 (only the most actionable)
```

### 2. Levels - Bullet Lists Only
```
No descriptions, just facts:
* Level * Level * Level

Critical level gets one-line explanation:
ES 6745 - Range support - defended 2x today

No zones, no test history, no verbose context
Just: Level - Role - Brief reason
```

### 3. Runners - One Line Each
```
Format: Entry → Current (+PL) | Stop | Target

Protection mode: ACTIVE or OFF with one-line rule
```

### 4. Scenarios - IF-THEN Only
```
Two paths max:
* Hold path
* Lose path

One "Expect" line (10 words max)
```

### 5. Yesterday - Single Line
```
Setup @ Level: Result (outcome) | Learn: insight

No play-by-play, no timestamps, no detail
Just: What happened + what to learn
```

### 6. What to CUT (vs Full Version)
```
- Acceptance pattern diagrams
- "Prerequisites checklist"
- Multiple entry alternatives  
- Management protocol steps (assumed known)
- Context paragraphs
- Extended quotes
- Invalidation sections (implied in stop)
- Timing guidance (implied in execution)
- Minor support/resistance
- Decision trees (simplified to two paths)
- All educational content
- Example acceptance patterns
- Volatility explanations
- Quality checklists
```

### 7. What to KEEP (vs Full Version)
```
- All Trade Ideas for the next session
- All Failed Breakdown levels
- Acceptance duration (number only)
- Entry/Stop/Targets with R:R
- Key warnings (one per setup)
- Current runners with stops
- Protection mode status
- Critical level explanation
- Golden Rule application (in TL;DR)
- Yesterday's key learning
```
